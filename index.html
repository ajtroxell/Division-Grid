Division

Download Division 2.0!

Division isn't anything revolutionary when it comes to grids or grid customization. Inspired indirectly by Foundation, yet designed with the undesigner in mind. Division is responsive with mobile specific options.

There are a couple major differences between Division and a grid like Foundation or Twitter Bootstrap. First is our naming convention, which we set up for lightning fast prototyping. You no longer have to be concerned with how many columns an element must be in order to be two thirds, one third, or three-fifths. This is an excellent starting point for designers who are doing rapid prototyping of a concept before handing it over to developers.

Second is the amount of clutter. We run a very clean shop in comparison to other popular grids. While using a simple CSS reset, thanks to Eric Meyer, we keep the rest of our grid to a minimum. You don't need to "undo" styles in order to apply your own. Especially to elements like forms, lists, buttons, and blockquotes. Your design is yours, and we want it to stay that way. Our grid weighs in at only 8KB, 12KB including the CSS reset.

We have also included Respond.js, a great script for adding CSS3 Media Query support to older versions of IE. Respond.js is fully compatible with Division, unlike with some other grids.

Implementation is simple. Copy division.css or division.min.css as well as reset.min.css into your Stylesheet directory, and then paste the code below all of your CSS files the head section of your site.

            <link rel="stylesheet" href="STYLESHEET DIRECTORY/reset.min.css">
            <link rel="stylesheet" href="STYLESHEET DIRECTORY/division.min.css">

Copy respond.min.js into your Javascript directory, and paste the code below into your footer.

            <script type="text/javascript" src="JAVASCRIPT DIRECTORY/respond.min.js"></script>

Nesting

Nesting is unlimited in depth and will always produce the correct proportions. Simple start a new div class="row" for each new set of nested divisions. Mobile divisions allow you to maintain a grid-like structure even on a mobile device.

Left & Right

Lefts and rights allow you to position divisions to the left or right of each other while still maintaining the same linear code structure. If an item appears first in your code, on mobile it will display first, but using lefts and rights allow you to position items in a different order on desktops and tablet devices.

Mobile

Division is supported even on mobile devices. Even though many grids have a mobile specific grid, we wanted to make it simpler to apply this to a grid. By giving any Division element an additional class of "mobile" you ensure any grid maintains it width no matter what device it is viewed on.
